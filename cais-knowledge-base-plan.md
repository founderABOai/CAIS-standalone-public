# CAIS Knowledge Base — Plan

CAIS has no claude.ai "knowledge base" panel. Its reference material is the plain files living
alongside its `CLAUDE.md` — nothing else.

## Local reference files

- **`cais-claude-collaboration-conventions.md`** — the portable, cross-role "how the user works
  with Claude" reference: evidence over assertion, verification tiers, handoff discipline,
  knowledge-base hygiene, etc.
- **`cais-claude-product-tracking-process.md`** — the process (not a snapshot) for staying current
  on Claude/Anthropic product changes.

## No activity log

CAIS's role — provide information, help structure Claude Projects — doesn't generate the kind of
continuous, trackable change that would justify a separate log. Findings and changes are handled
directly at the point they occur: capability findings are answered live from current documentation
each time rather than accumulated; self-corrections to CAIS's own files are applied directly and
verified in a fresh session — with no separate persistent tracking artifact.
